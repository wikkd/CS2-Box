## 1. 当前 UI 问题分析

### 1.1 布局与视觉

| # | 问题 | 严重程度 | 详细说明 |
|---|------|---------|---------|
| L1 | **全部使用百分比硬编码定位** | 🔴 严重 | `this.width * 37 / 100` 等遍布全文件，无组件化抽象，修改布局需逐个调整魔法数字 |
| L2 | **无清晰的视觉分区** | 🔴 严重 | 六个功能区域（Header / 基本信息 / 实体 / 权重 / 物品池 / 按钮）仅靠位置区分，无边框、卡片或色块分隔 |
| L3 | **信息密度过高** | 🟡 中等 | 1550 行代码全部塞入单屏幕，实体列表 + 权重 + 物品池 + 按钮全部堆叠，滚动体验差 |
| L4 | **字体缩放不一致** | 🟡 中等 | 混用 0.5F / 0.55F / 0.6F / 0.65F / 0.7F / 0.8F / 1.2F / 2.0F 共八种缩放级别，视觉层次混乱 |
| L5 | **低对比度配色** | 🟡 中等 | `0xFFD3D3D3` 文字配 `0xFF333333` 背景 —— 在低亮度屏幕上可读性差，无高对比度模式 |
| L6 | **按钮样式原始** | 🟢 轻微 | 纯色填充矩形 + 1px 边框，无圆角、阴影、渐变或 hover 动画过渡 |

### 1.2 交互与可用性

| # | 问题 | 严重程度 | 详细说明 |
|---|------|---------|---------|
| I1 | **实体搜索无实时建议** | 🔴 严重 | `doEntitySearch()` 在 `charTyped` 中触发，但搜索结果无分类、无图标、无高亮匹配文本 |
| I2 | **物品添加仅支持手动输入 ID** | 🔴 严重 | `addItemById()` 要求玩家手动输入完整 ResourceLocation（如 `minecraft:diamond_sword`），极易出错 |
| I3 | **无拖拽支持** | 🔴 严重 | 物品不能跨品质拖拽移动，实体列表不支持重排，所有操作依赖点击 |
| I4 | **仅支持 Ctrl+Z 撤销，无 Ctrl+Y 重做** | 🟡 中等 | 撤销后无法恢复，线性历史不支持分支 |
| I5 | **滚动区域过多且独立** | 🟡 中等 | 实体列表（含横向滚动）、物品列表、弹窗搜索列表 —— 共 4 个独立滚动区，鼠标滚轮焦点混乱 |
| I6 | **删除操作无批量能力** | 🟡 中等 | 删除物品/实体需逐一点击，无法多选后批量删除 |
| I7 | **实体添加时掉落率不可调节** | 🟡 中等 | `EntityDropEntry(id, 1.0F)` 写死 100%，添加后无 UI 可修改 |
| I8 | **只读模式提示微弱** | 🟢 轻微 | 仅右上角一行红色小字，无视觉降级（灰显、锁定图标） |
| I9 | **Esc 键行为不一致** | 🟢 轻微 | 在弹窗中按 Esc 关闭弹窗，在主界面按 Esc 却触发"放弃更改"确认而不是返回上级 |

### 1.3 功能缺失

| # | 问题 | 严重程度 |
|---|------|---------|
| F1 | **无物品搜索/浏览** —— 无法像创造模式那样浏览所有注册物品 | 🔴 严重 |
| F2 | **无品质间物品复制/移动** —— 同物品配不同品质需手动添加两次 | 🔴 严重 |
| F3 | **无箱子的 3D 预览** —— 编辑时看不到箱子的游戏内外观 | 🟡 中等 |
| F4 | **无 NBT/组件查看器** —— 编辑时无法查看物品的 data components | 🟡 中等 |
| F5 | **无导入/导出** —— 单个箱子不能导出为 JSON 文件供分享 | 🟢 轻微 |
| F6 | **无箱子复制** —— 不能基于现有箱子快速创建变体 | 🟢 轻微 |
| F7 | **无操作历史面板** —— 只能 Ctrl+Z 一步步撤销，无历史列表 | 🟢 轻微 |

### 1.4 BoxListScreen 问题

| # | 问题 |
|---|------|
| BL1 | 纯文本列表 —— 无法快速识别箱子（无图标/颜色预览） |
| BL2 | 无搜索/排序 —— 箱子多时难以快速定位 |
| BL3 | 无右键菜单 —— 删除需进入编辑界面 |
| BL4 | 新建箱子强制打开 ID 输入弹窗 —— 打断浏览流程 |

## 2. 重设计总体架构

### 2.1 设计原则

1. **分而治之**：将 1550 行单文件拆分为多个可组合的 UI 组件
2. **左侧导航 + 右侧内容**：类似 VS Code 的侧边栏布局，减少滚动
3. **所见即所得**：物品选择器使用创造模式物品栏式的图标网格
4. **渐进式复杂度**：常用操作一键可达，高级操作通过右键菜单/快捷键访问
5. **视觉一致性**：统一的配色、圆角、间距、字体层级

## 3. 组件树与数据流

### 3.1 数据模型 (不变)

```
BoxDefinition (Record, Immutable)
├── id: ResourceLocation
├── name: Component
├── keyItem: ResourceLocation
├── dropRate: float
├── dropEntities: List<ResourceLocation>
├── grades: List<GradeGroup>
│   ├── id: String
│   ├── displayName: String
│   ├── color: int
│   ├── weight: int
│   └── items: List<ItemStack>
├── texture: Optional<ResourceLocation>
├── sound: Optional<ResourceLocation>
└── entityDropRates: Map<ResourceLocation, Float>
```

### 3.2 编辑状态 (Mutable Copy)

```
EditorState (new, mutable)
├── boxId: String
├── name: String
├── dropRate: float
├── keyItem: ResourceLocation
├── entities: List<EntityEntry>
│   ├── entityId: ResourceLocation
│   └── dropRate: float
├── grades: List<GradeState>
│   ├── weight: int
│   └── items: List<ItemStack>
├── texture: Optional<ResourceLocation>
├── sound: Optional<ResourceLocation>
├── dirty: boolean
└── undoStack: List<EditorSnapshot>
```

### 3.3 数据流

```
                    ┌──────────────┐
                    │  JSON Files   │
                    │ config/csbox/ │
                    └──────┬───────┘
                           │ 启动加载 / reload
                           ▼
┌──────────┐       ┌──────────────┐       ┌───────────────┐
│  KubeJS  │──────▶│ BoxRegistry  │◀──────│ /csbox 命令    │
│  Scripts │       │ (LinkedHM)   │       │ add/set/give   │
└──────────┘       └──────┬───────┘       └───────────────┘
                           │ get(id)
                           ▼
                  ┌─────────────────┐
                  │  BoxDefinition   │  (Immutable Record)
                  └────────┬────────┘
                           │ from(def)
                           ▼
                  ┌─────────────────┐
                  │  EditorState     │  (Mutable, GUI works on this)
                  │  · snapshot()    │
                  │  · undo()        │
                  │  · redo()        │  ← NEW
                  │  · toDefinition()│
                  └────────┬────────┘
                           │ 用户编辑
                           ▼
                  ┌─────────────────┐
                  │  用户点击 Save    │
                  └────────┬────────┘
                           │
              ┌────────────┴────────────┐

    ┌─────────────────┐      ┌─────────────────┐
```

## 5. 交互流程优化

### 5.1 键盘快捷键

| 快捷键 | 上下文 | 功能 |
|--------|--------|------|
| `Ctrl+S` | 全局 | 保存当前箱子 |
| `Ctrl+Z` | 全局 | 撤销 |
| `Ctrl+Y` | 全局 | **重做（新增）** |
| `Ctrl+F` | 任何列表 | 聚焦搜索框 |
| `Ctrl+A` | 物品网格 | 全选当前品质物品 |
| `Ctrl+D` | 物品网格 | **复制选中物品到当前品质** |
| `Delete` | 列表/网格 | 删除选中项 |
| `Escape` | 弹窗 | 关闭弹窗 |
| `Escape` | 主界面 | **返回上级（不再弹确认）** |
| `Tab` | 表单 | 切换输入字段 |
| `1`-`5` | 全局 | **快速切换到对应品质面板** |
| `Ctrl+1~5` | 物品已选中 | **移动选中物品到品质1-5** |
| `↑↓` | 列表 | 导航列表项 |
| `Enter` | 列表 | 确认选择 |

### 5.2 拖拽支持

| 拖拽操作 | 效果 |
|----------|------|
| 物品槽 → 另一个品质的导航按钮 | 移动物品到目标品质 |
| 物品槽 → 同一品质网格的其他位置 | 重排物品顺序 |
| Ctrl+拖拽 物品槽 → 另一个品质 | **复制**物品到目标品质 |
| 实体卡片 ↔ 实体卡片 | 重排实体顺序 |
| 物品从背包 (外部) → 品质面板 | 添加物品（需要特殊处理） |

### 5.3 搜索/筛选

- **实体搜索**: 实时过滤（每次输入更新），支持中文名、英文名、ResourceLocation
  - 分类过滤标签: 全部 / 被动 / 敌对 / 中立 / Boss / 载具 / 其他
  - 结果最大 50 条，按匹配度排序（精确匹配 > 前缀匹配 > 包含匹配）
  
- **物品搜索**: 类似创造模式物品栏
  - 输入即过滤，支持物品名、ID、创造模式标签
  - 分类标签: 全部 / 武器 / 装备 / 工具 / 方块 / 食物 / 杂项
  - 搜索结果以网格展示，hover 显示 tooltip

- **箱子列表搜索**: 按名称或 ID 过滤
  - 排序: 名称 ↑↓ / 物品数 ↑↓ / 掉落率 ↑↓

### 5.4 批量操作

- **多选模式**: 
  - `Ctrl+Click`: 追加选择
  - `Shift+Click`: 范围选择
  - `Ctrl+A`: 全选当前品质

- **批量操作菜单** (选中 ≥2 个物品时显示):
  - 移动到品质 X
  - 复制到品质 X
  - 删除选中
  - 设置数量为 N
  - 全部设为最大堆叠

### 5.5 撤销/重做增强

```
当前: [state_snap_0] ← [state_snap_1] ← [state_snap_2] ← (current)
                                                             ↑ undoIndex

新增: 支持重做 (Ctrl+Y)
      undoIndex 后退时不清除后续历史，允许前进
      
      最大历史: 50 步 (从 20 提升)
      历史面板: 可选的侧边栏列表，显示每步操作的描述
```

## 6. 与现有系统的数据联动

### 6.1 KubeJS 集成

```
KubeJS 脚本 (startup_scripts)
    │
    CsboxEvents.registry(event => {
        event.create("my_box", "我的箱子")
            .key("csgobox:csgo_key0")
            .dropRate(0.15)
            .dropFrom("minecraft:zombie")
            .addGrade(grade => grade
                .name("custom_grade")
                .color(0xFF00FF)
                .weight(10)
                .addItem("minecraft:diamond", 1)
            )
    })
    │
    ▼
BoxRegistry.register(boxDef)
    │
    ▼
GUI BoxListScreen 自动显示新箱子
```

**数据一致性保证**:
- KubeJS 在模组启动时注册箱子，JSON 文件在 `/csbox reload` 时加载
- 编辑器保存后 → JSON 写入 → `BoxRegistry` 更新
- KubeJS 注册的箱子 **不在** `config/csbox/` 中存 JSON（避免覆盖脚本定义）
  - 但编辑器可以 **查看** KubeJS 箱子（只读模式）
  - 通过 `BoxDefinition.source()` 或标记字段区分来源

### 6.2 /csbox 命令联动

```
/csbox editor  →  PacketOpenBoxEditor → Client: open BoxListScreen
/csbox gui     →  PacketOpenConfigPanel → Client: open Cloth Config
/csbox list    →  列出所有箱子 (无变化)
/csbox add     →  命令行添加物品 (无变化，但 GUI 更加方便)
/csbox give    →  给予箱子物品 (无变化)
/csbox reload  →  重新加载 JSON 配置 (无变化)

新增命令:
/csbox export <boxId>  →  导出箱子 JSON 到聊天栏或剪贴板
```

### 6.3 网络数据包

```
现有 (保持不变):
  PacketSyncBoxSave     C→S  保存箱子定义
  PacketSyncBoxDelete   C→S  删除箱子
  PacketOpenBoxEditor   S→C  打开编辑器
  PacketRequestBoxItems C→S  请求箱子物品数据
  PacketSyncBoxItems    S→C  同步箱子物品
  PacketBoxOpenResult   S→C  开箱结果
  PacketGiveItem        C→S  请求给予物品
  PacketCsgoProgress    C→S  同步进度种子

新增:
  PacketSyncBoxDuplicate C→S  复制箱子 (NEW)
  PacketRequestExport    C→S  请求导出箱子 JSON (NEW)
```
