# Technology Stack

**Analysis Date:** 2026-06-29 (incremental update; baseline 2026-06-28)

## Languages

**Primary:**
- Java 25 (toolchain-pinned via `gradle.properties` -> `org.gradle.java.home=/Library/Java/JavaVirtualMachines/jdk-21.jdk/Contents/Home`, while `build.gradle` declares `JavaLanguageVersion.of(25)`) - All sources under `src/main/java/com/reclizer/csbox/**`

- TOML - runtime configuration emitted to `config/csbox-common.toml`

**Toolchain notes (from `gradle.properties` + `build.gradle`):**
- `org.gradle.java.home` pinned to a local macOS JDK 21
- Java toolchain in `build.gradle` requests `JavaLanguageVersion.of(25)` and the NeoGradle 7.1.38 recompiler is launched with `--enable-preview` to handle unnamed `_` variables introduced by MC 26.1.2
- See `docs/port-26.1.2.md` for the rationale (env requirements shifted from Java 21 -> 25 for the 26.1.2 port)
- The `runClient` run config is forced to JDK 21 by the Gradle toolchain specifier (commit a8bea6a, 2026-06-29) so the `runClient` JVM and the Gradle daemon use the same JDK regardless of the shell's `JAVA_HOME`

## Frameworks

**Core (mod loader / Minecraft):**
- NeoForge 26.1.2.76 (API entrypoint `CsBox.java` annotated `@Mod(CsBox.MODID)`)
- Minecraft 26.1.2 (vanilla jars obtained via NeoGradle's NeoForm 26.1.2 pipeline)
- Forge FML event bus (`net.neoforged.bus`, `net.neoforged.fml`)
- NeoForge network payload bus (`net.neoforged.neoforge.network`)

**GUI / Rendering:**
- Minecraft client rendering APIs (no external GUI library; the mod's screens extend `net.minecraft.client.gui.screens.Screen` directly)
- 2D matrix stack `org.joml.Matrix3x2fStack` (provided transitively by Minecraft/NeoForge)

**Persistence / Config:**
- NeoForge `ModConfigSpec` (TOML-backed config system) - sole config backend since v1.0.5 (Cloth Config removed)
- `CsboxConfig` lives in the `com.reclizer.csgobox.config` package (relocated from `com.reclizer.csbox.config` in commit b7b11e5, 2026-06-29)
- JSON via `com.google.gson` (Gson) for box definition files
- Mojang `DataComponentPatch` + `JsonOps`/`NbtOps` codecs for item component data

**Command system:**
- Mojang `brigadier` (`com.mojang.brigadier.*`) - core Minecraft dependency, used in `CsboxCommand.java`

**Build/Dev:**
- NeoGradle 7.1.38 userdev plugin (compiles Minecraft + NeoForge sources, then project sources into a single jar)
- `datagen` run config is intentionally commented out in `build.gradle` (lines 41-43) for 26.1.2

**Testing (project-side):**
- None - no test framework is configured. NeoForge's vanilla game-test framework is enabled via `forge.enabledGameTestNamespaces=mod_id` for `client`, `server`, `gameTestServer` runs, but the project does not ship any tests of its own.

## Configuration

**Build:**
- `gradle.properties` - version pins (`minecraft_version`, `neo_version`, `mod_version`, `mod_id`, `mod_group_id`, `mod_name`, `mod_authors`, `mod_description`, mapping channel/version, Java home)
- `settings.gradle` - plugin versions (NeoGradle userdev 7.1.38, foojay-resolver 0.9.0)
- `build.gradle` - task wiring (`processResources` placeholders, `runs` configurations, repository declarations, jar manifest). The `runClient` runs block pins the toolchain to JDK 21 (commit a8bea6a) so launching the dev client always uses JDK 21.
- Resource expansion: `META-INF/neoforge.mods.toml` and `pack.mcmeta` are filtered through `replaceProperties` at `processResources` time

**Mod (runtime, TOML):**
- NeoForge `ModConfigSpec` registered in `CsBox.java:45` -> `config/csbox-common.toml` (file name forced by the third arg)
- Section grouping: `[general]`, `[advanced]`, `[sound]`, `[animation]` (see `CsboxConfig.java` in the `com.reclizer.csgobox.config` package)
- Section comments are emitted in the TOML, but Java callers access fields flatly (`CONFIG.fieldName`, no `CONFIG.section.fieldName`) - documented in `AGENTS.md`
- Field defaults (v1.0.5):
  - `animationSpeed=NORMAL`, `globalDropRatePercent=100`
  - `loadDefaultBoxes=true`, `enableDebugLogging=false`
  - `openSoundVolume=100`, `tickSoundVolume=50`, `finishSoundVolume=100`
  - `totalAnimationTicks=145`, `animationSpeedMultiplier=1`, `showItemNames=true`

**Mod (data):**
- `config/csbox/*.json` - one file per box definition (created on demand)
- `src/main/resources/assets/csbox/lang/{en_us,zh_cn}.json` - localization
- `src/main/resources/data/csgobox/recipe/*.json` - recipe definitions (3 crafting_shaped + 1 smithing_transform) — directory renamed from `recipes/` (plural) to `recipe/` (singular) on 2026-06-29 to match the Minecraft data-pack convention; `RecipeManager` resolves the singular name via `Registries.elementsDirPath(Registries.RECIPE)`. The netherite workbench recipe for `csgo_key3` was removed in v1.0.5 (commit be40e5a) so `csgo_key3` is now obtainable only via the smithing table recipe `csgo_key3_smithing.json`.

**Java compiler flags:**
- Source encoding: `UTF-8`
- Preview features enabled (`--enable-preview`) for NeoGradle recompiler and Java compile (see `gradle.properties` lines 5-6)

## Platform Requirements

**Development:**
- JDK 25 toolchain (declared) with JDK 21 also available (`gradle.properties` pins `org.gradle.java.home` to a local JDK 21 for the Gradle daemon)
- The `runClient` run configuration is explicitly pinned to JDK 21 via the Gradle toolchain specifier in `build.gradle` (commit a8bea6a, 2026-06-29) — this overrides the shell's `JAVA_HOME` and guarantees the dev client JVM matches the Gradle daemon's JDK 21
- Gradle 8.14 (wrapper-managed)
- Internet access for Maven repos (Maven Central, NeoForged releases, CurseMaven, Latvian, Modmaven, BlameJared)
- macOS, Linux, or Windows (no platform-specific code; verified builds on macOS per `gradle.properties`)
- Disk: roughly 2-3 GB for NeoForm decompile + recompile output (`build/neoForm/`)
- Ram: `org.gradle.jvmargs=-Xmx3G` (`gradle.properties` line 1)

**Production (deployment):**
- Minecraft 26.1.2 client or server with NeoForge 26.1.2.76+ installed
- Side: `BOTH` (per `neoforge.mods.toml` - the mod loads on dedicated servers, integrated servers, and clients)
- Built jar is produced at `build/libs/csbox-<mod_version>.jar` (currently `csbox-1.0.5-26.1.2.jar`)
- Manifest attributes baked into the jar: `Specification-Title`, `Specification-Vendor`, `Specification-Version`, `Implementation-Title`, `Implementation-Version`, `Implementation-Vendor`, `Implementation-Timestamp`
- No services registration (no `META-INF/services/...`)

## Update 2026-06-29
