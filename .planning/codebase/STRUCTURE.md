## Directory Purposes

**`src/main/java/com/reclizer/csgobox/`:**
- Purpose: All mod Java source.
- Contains: 14 subpackages, 36 source files.
- Key files: `CsgoBox.java` (entry point).

**`src/main/java/com/reclizer/csgobox/box/`:**
- Purpose: Box domain model (definition, grade, registry, JSON loader).
- Contains: `BoxDefinition.java`, `BoxRegistry.java`, `BoxJsonLoader.java`, `GradeGroup.java`.

**`src/main/java/com/reclizer/csgobox/capability/`:**
- Purpose: Per-player persistent state via NeoForge attachments.
- Contains: `CsboxPlayerData.java`, `ModCapability.java`.

**`src/main/java/com/reclizer/csgobox/command/`:**
- Purpose: Operator command tree (`/csbox ...`).
- Contains: `CsboxCommand.java`.

**`src/main/java/com/reclizer/csgobox/advancement/`:**
- Purpose: Custom criterion triggers for the advancement system.
- Contains: `OpenedBoxTrigger.java`, `ModLoadedTrigger.java`.

**`src/main/java/com/reclizer/csgobox/event/`:**
- Purpose: NeoForge event bus subscribers (mob death, right-click, player login).
- Contains: `ModEvents.java`, `ClickEvent.java`.

**`src/main/java/com/reclizer/csgobox/gui/`:**
- Purpose: Client-only `Screen` subclasses.
- Contains: `CsboxScreen.java`, `CsboxProgressScreen.java`, `CsLookItemScreen.java`.

**`src/main/java/com/reclizer/csgobox/item/`:**
- Purpose: Item types and their registration.
- Contains: `ItemCsgoBox.java`, `ItemCsgoKey.java`, `ModItems.java`.

**`src/main/java/com/reclizer/csgobox/packet/`:**
- Purpose: `CustomPacketPayload` records + shared validation.
- Contains: `PacketBoxOpenResult.java`, `PacketRequestBoxItems.java`, `PacketSyncBoxItems.java`, `PacketCsgoProgress.java`, `PacketValidation.java`.

**`src/main/java/com/reclizer/csgobox/sounds/`:**
- Purpose: Sound event registration.
- Contains: `ModSounds.java`.

**`src/main/java/com/reclizer/csgobox/utils/`:**
- Purpose: Pure helpers (no I/O, no game state mutation in most cases).
- Contains: `RandomItem.java`, `IconListTools.java`, `GuiItemMove.java`, `RenderFontTool.java`, `ColorTools.java`, `OverlayColor.java`, `EntityChineseMap.java`.

**`src/main/java/com/reclizer/csgobox/config/`:**
- Purpose: ModConfigSpec config class.
- Contains: `CsboxConfig.java`.

**`src/main/resources/META-INF/`:**
- Purpose: Mod loader metadata.
- Contains: `neoforge.mods.toml` (template-tokens substituted at build time).

**`src/main/resources/assets/csgobox/`:**
- Purpose: Client-side mod assets.
- Contains:
  - `lang/en_us.json`, `lang/zh_cn.json` - translation keys.
  - `sounds.json` - sound registry.
  - `textures/item/csgo_box.png`, `csgo_key0..3.png` - item textures.
  - `textures/screens/csgo_background.png`, `gold_item.png` - GUI textures.
  - `textures/gui/advancements/backgrounds/background_root.png` - advancement tab background (new in v1.0.5).
  - `models/item/csgo_box.json`, `csgo_box_test.json`, `csgo_key0..3.json` - item models.

**`src/main/resources/assets/minecraft/`:**
- Purpose: Override vanilla-namespace resources.
- Contains: `shaders/post/fade_in_blur.json` + `shaders/program/fade_in_blur.json` (registers a custom shader post-effect; no `.glsl` source detected, so this is a placeholder/shim).

**`src/main/resources/data/csgobox/`:**
- Purpose: Server-side data (advancements + recipes).
- Contains:
  - `advancement/root.json` (uses `csgobox:mod_loaded` trigger, chest icon).
  - `advancement/first_box.json` (uses `csgobox:opened_box`, no count - fires on first open).
  - `advancement/shopper.json` (uses `csgobox:opened_box` with `count: 200`, challenge frame, hidden).
  - `recipe/csgo_key0.json` (3 iron ingots -> key0).
  - `recipe/csgo_key1.json` (3 gold ingots -> key1).
  - `recipe/csgo_key2.json` (3 diamonds -> key2).
  - `recipe/csgo_key3_smithing.json` (smithing: diamond key + netherite ingot + netherite upgrade template -> key3). Note: `csgo_key3.json` (vanilla crafting recipe) is NOT present - smithing is the only path.

**`src/main/resources/pack.mcmeta`:**
- Purpose: Resource-pack descriptor.
- Contains: `pack_format: 34`, supported `[34, 48]`.

**`build/`:**
- Purpose: Generated by Gradle. Contains `build/libs/csgobox-1.0.5.jar`, `build/classes/`, `build/resources/main/` (the processed resource pack).
- Generated: Yes.
- Committed: No (in `.gitignore`).

**`src/generated/`:**
- Purpose: Generated by the `gradle runData` task. Where the NeoForged data generator dumps processed resource dumps.
- Generated: Yes.
- Committed: No.

**`runs/`:**
- Purpose: Gradle-userdev-generated run configurations for `runClient` and `runServer`.
- Generated: Yes.
- Committed: No (likely in `.gitignore`).

**`docs/`:**

## Key File Locations

**Entry Points:**
- `src/main/java/com/reclizer/csgobox/CsgoBox.java`: Mod entrypoint (`@Mod(CsgoBox.MODID)`). Registers everything in its constructor.

**Configuration:**
- `src/main/java/com/reclizer/csgobox/config/CsboxConfig.java`: `ModConfigSpec` definition. Sections: `[general]`, `[advanced]`, `[sound]`, `[animation]`. All values materialised in the constructor (no `init()`).
- `gradle.properties`: Build-time version pins.
- `build.gradle`: Build script.
- `src/main/resources/META-INF/neoforge.mods.toml`: Mod metadata (template-tokens replaced at build).
- `config/csgobox.toml` (generated at runtime): User-facing config written by NeoForge.
- `config/csbox/*.json` (generated/edited at runtime): Box definitions loaded by `BoxJsonLoader`.

**Core Logic:**
- `src/main/java/com/reclizer/csgobox/box/BoxDefinition.java`: Box spec (record + builder + stream codec).
- `src/main/java/com/reclizer/csgobox/box/BoxJsonLoader.java`: Filesystem loader/saver, default-box seeder.
- `src/main/java/com/reclizer/csgobox/packet/PacketCsgoProgress.java`: Server-authoritative box open roll (the heart of the mod).
- `src/main/java/com/reclizer/csgobox/utils/RandomItem.java`: Weighted-rarity selection algorithm.

**Testing:**
- `build.gradle:43-52`: GameTest framework is registered (`forge.enabledGameTestNamespaces=csgobox`) but no GameTest classes exist under `src/` - test infrastructure is enabled but unused.

## Naming Conventions

**Files:**
- Java classes: `PascalCase` (e.g., `BoxDefinition.java`, `PacketCsgoProgress.java`).
- JSON resources: `snake_case` for keys inside, filenames match their data id (e.g., `en_us.json`, `csgo_box.json`, `first_box.json`, `csgo_key0.json`).
- Textures: `snake_case` (e.g., `csgo_box.png`, `csgo_background.png`, `gold_item.png`).

**Java identifiers:**
- Classes: `PascalCase` (records use PascalCase too: `BoxDefinition`, `GradeGroup`, `CsboxPlayerData`, `PacketCsgoProgress`).
- Methods: `camelCase` (e.g., `getBoxId`, `tryConsumeKeys`, `consumeMatching`, `loadAll`).
- Constants: `UPPER_SNAKE_CASE` (e.g., `MAX_ITEMS`, `MAX_PENDING_RESULTS`, `OPENED_BOXES_STAT`).
- Enum constants: `UPPER_SNAKE_CASE` (`CsboxConfig.AnimationSpeed.SLOW`).
- Package names: lowercase single words (`box`, `item`, `gui`, `packet`, `event`, `command`, `sounds`, `utils`, `config`, `capability`, `advancement`).
- Resource location namespaces: `csgobox` (mod id) and `minecraft` (vanilla overrides only).

**Resource location conventions:**
- Mod id: `csgobox` (single token).
- Item ids: `csgobox:csgo_box`, `csgobox:csgo_key0..3`.
- Sound ids: `csgobox:cs_open`, `csgobox:cs_dita`, `csgobox:cs_finish`.
- Trigger ids: `csgobox:opened_box`, `csgobox:mod_loaded`.
- Custom stat id: `csgobox:opened_boxes`.
- Packet ids: `csgobox:csgo_progress`, `csgobox:box_open_result`, `csgobox:request_box_items`, `csgobox:sync_box_items`.
- Data component id: `csgobox:box_id`.
- Attachment type id: `csgobox:player_data`.
- Creative tab id: `csgobox:cs_tab` (used as both registry id and lang key prefix `itemGroup.csgobox.cs_tab`).

**Directories:**
- `src/main/java/<package-path>/`: standard Maven layout, package-by-feature.
- `src/main/resources/data/<namespace>/`: Minecraft data-pack convention (here only `data/csgobox/`).
- `src/main/resources/assets/<namespace>/`: Minecraft resource-pack convention.
- `config/`: written by ModConfigSpec at runtime (created by NeoForge, not present at build time).

## Where to Add New Code

**New box (no code change):**
- Drop a JSON file into `config/csbox/<your_box>.json`. Restart server (or `/csbox reload`). The file basename becomes the `ResourceLocation` (`csgobox:<basename>`). Use `/csbox give csgobox:<basename>` to issue.
- Shape reference: `BoxJsonLoader.writeDefaultIfEmpty` (`BoxJsonLoader.java:82-158`) is the canonical template.

**New item:**
- Add the `Supplier<Item>` to `ModItems.ITEMS` in `src/main/java/com/reclizer/csgobox/item/ModItems.java:46-51`.
- Create a `models/item/<id>.json` and `textures/item/<id>.png` under `src/main/resources/assets/csgobox/`.
- Add a lang entry to `assets/csgobox/lang/en_us.json` and `zh_cn.json`.
- If the item needs recipes, add JSON to `src/main/resources/data/csgobox/recipe/`.

**New GUI screen:**
- Add a new `extends Screen` class in `src/main/java/com/reclizer/csgobox/gui/`. Reuse existing utilities (`IconListTools`, `GuiItemMove`, `RenderFontTool`, `OverlayColor`).
- Trigger it from `ClickEvent` (right-click) or from another screen's `mouseClicked`/`tick`.

**New network packet:**
- Add a `record` implementing `CustomPacketPayload` in `src/main/java/com/reclizer/csgobox/packet/`.
- Define its `Type<>` and `STREAM_CODEC`.
- Run all fields through `PacketValidation` helpers in the compact constructor.
- Register in `CsgoBox.registerPayloads` (`CsgoBox.java:86-92`) using `registrar.playToServer` or `playToClient`.
- For server-side result delivery, queue the packet via `PacketValidation.trimQueue(sPendingResults, MAX_PENDING_RESULTS)` then `sPendingResults.add(...)`. Client screens consume via `consumeMatching(id)`.

**New criterion trigger:**
- Add a class in `src/main/java/com/reclizer/csgobox/advancement/` extending `SimpleCriterionTrigger<T extends SimpleInstance>`.
- Provide the `INSTANCE`, `ID`, and `TriggerInstance` record with `RecordCodecBuilder` `CODEC`.
- Register under `Registries.TRIGGER_TYPE` in `CsgoBox.java:67-75`.
- Fire from `IPayloadContext` handlers or event subscribers via `INSTANCE.trigger(serverPlayer)`.

**New config option:**
- Add a `ModConfigSpec.*Value` field in `CsboxConfig.java`. Place it in the right section (`builder.push("general")`, etc.).
- Use `builder.comment("...").define(...)` for booleans, `defineInRange` for numbers, `defineEnum` for enums.
- Call `.get()` directly inside an accessor method - no `init()` helper.
- Access via `CsgoBox.CONFIG.yourField()`.

**New utility:**
- Add a `final class` with private constructor in `src/main/java/com/reclizer/csgobox/utils/`. Match the package-by-feature convention.

**New advancement:**
- Add JSON to `src/main/resources/data/csgobox/advancement/`.
- Set `parent` to `"csgobox:root"` for descendants.
- Add `csgobox:opened_box` trigger with optional `conditions.count`.
- Add lang keys under `advancement.csgobox.<id>.title` and `.description`.

**New command subcommand:**
- Edit `src/main/java/com/reclizer/csgobox/command/CsboxCommand.java`. Append a new `.then(Commands.literal("yoursub")...)` to the existing dispatcher builder inside `register()`.
- Add corresponding help + lang entries.
