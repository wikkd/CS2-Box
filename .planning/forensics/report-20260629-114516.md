## Root Cause Hypothesis

The v1.0.5 release shipped broken because **no verification gate existed
between "code looks right" and "tagged as release"**. Specifically:

1. **No automated build.** `./gradlew build` would have failed to
   compile the missing `CsboxConfig` class.
2. **No smoke test.** No test that opens a CS:GO box client-side, which
   would have surfaced CR-002 (NPE) and CR-001/CR-003 (drop rate = 0).
3. **No recipe-load test.** No datapack-load smoke test that would have
   surfaced CR-004 (recipes/ vs recipe/).
4. **No doc/code coherence hook.** The recipe path rename in `24a82ac`
   silently left 3 doc files referring to the old path for 2h 39m.
5. **No formal release tagging.** Releases exist only as commits, with
   no immutable v1.0.5 reference for distribution.

The post-release hot-patch cluster (8 commits in 24h) is the symptom —
the author found and fixed everything they could *manually* find after
the release. The code review (`v1.0.5-REVIEW.md`, committed as
`5f3856a`) surfaced three more bugs that the manual pass missed
(CR-001/CR-002/CR-003), and these are **still unfixed on `main` HEAD**.
